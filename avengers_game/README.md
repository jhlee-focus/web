# Avengers Game Center

`index.html` 에서 출발하는 작은 마블 테마 게임 모음. 모두 정적 HTML 한 파일로 동작하며 GitHub Pages 에서 직접 서빙됩니다.

| 페이지 | 설명 |
|---|---|
| `index.html` | 🎮 게임 허브 (네 게임으로 가는 카드 그리드) |
| `memory-game.html` | 🃏 메모리(짝맞추기) — 31명 히어로 카드, 난이도 3단계 |
| `puzzle-game.html` | 🔲 슬라이딩 퍼즐 — 빈 칸 한 칸으로 이미지를 맞춤 |
| `drag-puzzle.html` | 🧩 직소 퍼즐 — 조각을 드래그해서 끼움 |
| `hero-run.html` | ⚡ 히어로 런 — 캔버스 액션 (달리기 모드 / 사격 모드) |

## 폴더 구조

```
avengers_game/
├── index.html
├── memory-game.html
├── puzzle-game.html
├── drag-puzzle.html
├── hero-run.html
├── README.md                      ← 이 파일
├── 2d_game_assets/                ← hero-run 전용 이미지 (방패/망치/캡틴 noshield 등)
├── images/
│   ├── images.js                  ← memory-game 의 이미지 목록 (수동 관리)
│   └── *.webp                     ← 31명 히어로 단일 캐릭터 이미지
└── puzzle_images/
    ├── images.js                  ← puzzle-game / drag-puzzle 의 이미지 목록 (수동 관리)
    └── *.webp / *.jpg / *.png     ← 단체 / 배경 이미지
```

## 이미지 목록 갱신

`images/images.js`, `puzzle_images/images.js` 는 **수동 관리**합니다. 새 이미지를 추가하거나 이름을 바꾼 뒤 같은 이름을 배열에도 넣어 주세요.

**예시 — `images/images.js` 에 새 캐릭터 추가**
```js
window.MCU_IMAGES = [
  "그루트.webp",
  "네뷸라.webp",
  …
  "새캐릭터.webp",          // ← 추가
];
```

확인 방법:
1. 브라우저에서 `memory-game.html` (또는 `puzzle-game.html` / `drag-puzzle.html`) 열기
2. 콘솔에 `⚠️ 이미지 목록을 불러오지 못했습니다` 경고가 없는지 확인
3. 카드 / 퍼즐 후보에 새 항목이 나오는지 확인

> 향후 자동 갱신이 필요하면 `images/` 와 `puzzle_images/` 폴더를 스캔해서 두 `images.js` 를 재생성하는 PowerShell 스크립트를 추가할 수 있습니다.

## hero-run 의 외부 자산

`hero-run.html` 은 캐릭터 스프라이트(걷기 애니메이션)를 **base64 로 인라인**해서 들고 다닙니다. 발사체 / 캡틴 noshield 스프라이트는 `2d_game_assets/` 의 외부 파일을 참조하지만, base64 인라인이 필요한 자산(예: 알파 마스크 픽셀 충돌에 쓰는 방패 / 망치)은 `file://` 환경에서도 same-origin 으로 동작하도록 본문에 직접 박혀 있습니다.

## 스모크 테스트 체크리스트 (수동)

새 변경을 머지하기 전 5분 안에 돌릴 수 있는 최소 체크리스트.

### 공통
- [ ] `index.html` 5개 카드가 모두 보이고, 각 카드 클릭으로 해당 게임이 열린다
- [ ] 각 게임 페이지 콘솔에 에러가 없다 (특히 빨간 에러)
- [ ] 모바일 가로/세로 회전 후 레이아웃이 잘리지 않는다 (특히 하단 safe-area)

### 메모리 게임
- [ ] 난이도 3개(쉬움/보통/어려움) 전환 시 카드 수가 바뀐다
- [ ] 같은 카드 두 장 매칭 시 점수 + 사라짐
- [ ] "다시 시작" 으로 게임이 리셋된다

### 슬라이딩 퍼즐
- [ ] 이미지 선택기(◀ ▶)로 이미지가 바뀐다
- [ ] 난이도 변경 시 그리드가 재구성된다
- [ ] "다시 섞기" 후 다시 풀 수 있다
- [ ] 힌트 토글이 동작한다
- [ ] **이미지 로드 실패 케이스**: 존재하지 않는 파일명을 `puzzle_images/images.js` 에 한 줄 추가해도 퍼즐판이 16:9 fallback 으로 정상 그려진다

### 직소 퍼즐
- [ ] 조각 드래그로 정확한 슬롯에 끼워진다
- [ ] 잘못된 위치에 놓으면 트레이로 되돌아간다
- [ ] 난이도 변경 / 다시 섞기 동작

### 히어로 런
- [ ] 시작 화면: 검색창 + ▶ 게임 시작 + 📖 히어로 도감 + 모드 탭(🏃/🔫)
- [ ] 시작 검색: "토르" / "토니 스타크" / "스타크" / "캡 틴 아 메 리 카" 모두 매칭
- [ ] 도감 진입 시 12명이 앞을 보고 걷는 애니메이션
- [ ] 도감 셀 클릭 → 큰 이미지 + 어린이용 설명 오버레이 (선택은 변경되지 않음)
- [ ] 도감 검색 → 셀로 스크롤 + 플래시 + 오버레이 자동 표시
- [ ] 달리기 모드: 빌런 박치기로 처치 (ultronbot/ultron 도 한 방)
- [ ] 사격 모드: 캐릭터별로 다른 발사체가 0.42s 마다 자동 발사
- [ ] 캡틴 방패 / 토르 망치 부메랑: 한 개만, 빌런 관통(3 dmg), 끝/장애물에서 반전, 귀환 후 재발사
- [ ] 캡틴 스프라이트가 방패 비행 중에는 noshield 로 교체됨
- [ ] 게임 오버 후 최고 점수 누적, "↻ 초기화" 로 0 으로 되돌릴 수 있음
- [ ] OSK 토글 시 OS 가상 키보드가 뜨지 않음 (모바일)

## 기여 / 변경 시 메모

- 모든 게임은 정적 HTML 한 파일이라 빌드 / 번들러가 없습니다. 직접 편집 → 저장 → 새로고침.
- 캐릭터 스프라이트(걷기 애니메이션) 시트는 96×96 셀, 3 frames × 4 dirs (DIR_DOWN=0/LEFT=1/RIGHT=2 만 사용, 4 번째 행은 미사용).
- `hero-run.html` 은 이미 ~2,000 줄이라 큰 리팩터링은 단계적으로(데이터부터 분리) 진행하는 것이 안전합니다.
